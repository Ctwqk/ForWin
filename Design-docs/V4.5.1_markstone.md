# ForWin V4.5.1 Markstone：残余设计收束

更新时间：2026-04-26

状态：V4.5.1 是 V4.5 后端验收闭环之后的设计收束层，只整理当前仍需要在 V4.5.x 口径内追踪的残余设计债。V4.6 三层知识系统以 [V4.6_knowledge_system.md](/home/taiwei/.codex/worktrees/6974/ForWin/Design-docs/V4.6_knowledge_system.md) 为准；V4.6.x 的 UI、dashboard、脚本型 Skill、native GraphDelta extractor、完整 World Studio 不纳入本文档执行范围。

## 1. 定位

V4.5.1 不新增大架构，不重开主链；它负责把 V4.5 已完成的后端 contract 与仍需收束的设计债分清楚：

- 已完成项继续以 `V4.5_markstone.md` 的 M1-M6 为验收口径。
- V4.5.1 只记录仍需要小步收束的后端 contract、迁移审计、文档口径和治理策略。
- V4.6 知识系统固定 `BookState DB Canon -> Obsidian Vault -> Karpathy-style LLM KB` 的权威关系；完整可视化、dashboard、脚本执行、native extractor 和更复杂交互产品形态留给 V4.6.x。

## 2. V4.5.1 已收束项

以下项已经在代码和文档中落地，并作为 V4.5.1 的当前基线：

- Legacy `SubWorld.metadata_json.region_drafts` 幂等提升为 `map_regions`，保留原草案，写入 promotion report。
- Legacy import response / migration report 增加 promoted、created、skipped、scope conflict、unresolved refs 等字段。
- Legacy region promotion 通过 `DecisionEvent` 记录 start / success / fail。
- Genesis `map_atlas.edges` 的跨 subworld edge 可生成显式 `InterSubWorldConnectionSpec` / `world_gate`；无显式 edge 时回退 `default_chain`。
- `MapEnsureResponse.summary` 记录 `interconnection_source`，用于区分 `atlas_edges` 与 `default_chain`。
- Movement reviewer 支持 reviewer-only `movement_policy`：`allowed_access_rule_ids`、`travel_time_multiplier_by_edge_type`、`team_speed_multiplier`。
- Movement reviewer 对未授权 access rule 产出 deterministic issue：`map_access_rule_unmet`。
- Audit closure 已补齐：`DecisionEvent` 写入层统一 redaction，BookState / map / legacy / movement reviewer 关键事件使用 `v4.5.1-audit` payload schema。
- writer/model 失败但没有 `WriterOutput` 时会 flush 失败 `PromptTrace`，并以 child `PROMPT_TRACE_RECORDED` 关联失败事件。
- redacted diagnostic artifact manifest 已接入 chapter ledger；task timeline / chapter ledger 追加 `stage_durations` 与 `operation_ids`。
- 文档 grep guard 已防止 BookState 追加 gate、map 无 route、arc expansion 未接入等旧表述回流。

## 3. V4.5.1 继续追踪的残余设计

这些不是 V4.5 阻塞项，但仍应在 V4.5.x 口径内继续收束；每项都必须保持后端 contract / 文档 / 测试优先，不进入 V4.6 产品 UI 范围。

### 3.1 历史设计文档兼容标注

目标：

- `V2_9_2.md` 中仍保留的旧 `SubWorld` 局部舞台语义，只能标注为 compatibility / historical baseline。
- 新开发引用地图和 SubWorld 时，默认跳转到 `map_scheme_c.md` 与本文件。

验收：

- 活跃文档不再把城市、宗门、遗迹入口描述为新建 `SubWorld` 的目标对象。
- 任何 legacy `SubWorld` 迁移说明都指向 MapNode / site_state / region promotion，而不是创建新 SubWorld。

### 3.2 Genesis map_atlas 到 BookMap 的后端合并 contract

目标：

- 明确 Genesis `map_atlas.submaps / regions / nodes / edges` 到 BookMap 的后端合并规则。
- 明确重复导入、增量补齐、source id 保留和冲突报告的稳定字段。
- 保持 UI merge/editor 不在 V4.5.1 范围内。

验收：

- map ensure / arc expansion 产生的 summary 能解释新增 subworld、跳过 existing subworld、interconnection source。
- source ids 能在 MapNode / MapEdge / MapRegion metadata 中追溯。
- 无法自动解决的名字、scope 或 location ref 冲突进入 report，不静默生成错误对象。

### 3.3 Migration audit 最小闭环

目标：

- Legacy import、region promotion、site_state binding 的报告字段保持稳定。
- 失败时能通过 `DecisionEvent` payload 定位是 schema、promotion、binding 还是 scope conflict。

验收：

- migration report 至少包含：`created_site_state_map_bindings`、`promoted_region_draft_count`、`created_region_ids`、`skipped_region_drafts`、`scope_conflicts`、`unresolved_location_refs`。
- 重复运行 legacy import 不重复创建 map regions 或 site_state bindings。
- 城市、宗门、遗迹不会被 legacy import 误建为 `SubWorld`。

### 3.4 Observability contract hardening

状态：Audit First 后端闭环已完成；V4.5.1 不继续追踪 dashboard / UI 类观测产品化。

已完成：

- 对已有 `DecisionEvent` payload 做字段口径收束，优先覆盖 BookState compile/review、legacy projection、map generation/expansion、map movement review、legacy region promotion。
- 只补后端 trace completeness、payload schema 和 redaction 约束，不做 dashboard。
- 对 V3.8 历史复盘中的剩余后端缺口继续收束：failure prompt trace flush、raw/redacted artifact manifest、stage duration aggregation、operation correlation parity。

验收状态：

- 单章失败能从 task timeline 定位到 BookState、map、reviewer、legacy projection 或 region promotion。
- payload 中不写入完整正文、完整 prompt 或 hidden writer 不应见的 objective route。
- provider 400、parse failure、schema mismatch 等失败能留下 artifact URI / hash / summary 或等价审计 payload。
- 没有 writer output 的 LLM 失败调用也能保留失败 trace 或等价 DecisionEvent，避免只剩 runtime log。
- queue/context/LLM/review/repair/canon/cleanup/db-lock 等耗时字段可通过后端事件或只读 API 汇总。

仍不纳入 V4.5.1：

- 前端 Task Timeline / Chapter Ledger / LLM Trace Viewer、Failure Explorer、Governance / Audit Timeline、metrics dashboard。

### 3.5 Skill governance 后端口径

目标：

- 只收束 prompt-layer Skill 的项目级启用、禁用、strictness 和 trace 口径。
- 不做 Skill API / UI，不做脚本型 Skill，不引入 tool-backed execution。

验收：

- PromptTrace 能解释本次 writer/reviewer/genesis 使用了哪些 skill layer。
- 项目级 policy 与运行时选择不绕过 BookState canon gate。
- Skill strictness 的语义写入文档，避免与脚本执行能力混淆。

### 3.6 Movement policy v1 文档化

目标：

- 固化 reviewer-only `movement_policy` 的字段语义和 issue code。
- 保持 writer context 只看到 visible graph，不泄露 hidden objective route。

验收：

- `allowed_access_rule_ids` 缺失时不启用 access 限制；存在时未授权 route 返回 `map_access_rule_unmet`。
- `travel_time_multiplier_by_edge_type` 与 `team_speed_multiplier` 只影响 reviewer travel-time budget 计算，不改写 map canon。
- hidden / blocked / false route / known detour issue code 保持 deterministic。
