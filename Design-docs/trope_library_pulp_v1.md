# Trope Library V1 · Pulp Seed

更新时间：2026-05-18
状态：seed-content；待 `pulp_profile_upgrade_plan.md` Phase 5 MD loader 与 schema 扩展落地后即可加载
加载方式：`FORWIN_TROPE_TEMPLATE_PATH=Design-docs/trope_library_pulp_v1.md`

## 总则：四段式

所有 sub-trope 的正文交付必须遵循同一结构：

```
欲望建立 → 阻力加压 → 爽点兑现 → 余波钩子
```

每一段都是 writer 的硬约束，不是文学修辞。selector 选中某个 trope 后，把这 4 段文本直接注入 chapter plan 的对应 slot，writer 按段写。

## Schema 字段说明

每个 sub-trope 是一个 H2 section，标题格式：

```
## {template_id} · {display_name}
```

紧跟一个属性列表（被 md_loader 抓为 `TropeTemplate` 字段）：

| 字段 | 含义 | 取值 |
|---|---|---|
| `category` | 一级 reward tag | `power / social / justice / mystery / emotion` |
| `subcategory` | 二级爽点名 | 中文短语 |
| `market_tier` | 市场定位 | `sinking / mainstream / premium` |
| `cost_weight` | 实施成本（影响 selector 偏好） | `1`=单章兑现/便宜，`2`=需要一点铺垫，`3`=多章 setup/贵 |
| `best_window` | 何时用 | 逗号分隔标签：`band_early / band_mid / band_late / after_humiliation / after_low / public_scene / private_scene` |
| `genre_fit` | 适配题材 | 逗号分隔：`玄幻 / 都市 / 神豪 / 末世 / 种田 / 职场 / 宫斗 / 年代 / 萌宝` |

接下来 H3 段落顺序必须为：`欲望建立 / 阻力加压 / 爽点兑现 / 余波钩子 / anti_patterns / review_signals`。

## 通用反模式（全 trope 适用）

writer 在任何 sub-trope 下都不得：

- 用旁白宣布"爽"（"众人震惊"、"大家被打脸"等空泛词）
- 让主角全程被动，靠路人或系统白给
- 兑现没有客观证据（无文件、无动作、无效果）
- 让反派无理由突然降智
- 一次爽点解决整条主线（每次只解决当前小问题）
- 连续 ≥ 3 章重复同一种 sub-trope
- 章末没有钩子

## Selector 概览

`forwin/experience/trope_selector.py` 按以下信号选 trope：

1. `target_category`（来自 `BandExperienceScheduler`）→ 限定一级类
2. `band_position` ∈ [0, 1] → 匹配 `best_window`
3. `reward_gap` ≥ 2 → 强制选；reward_gap = 0 可跳过
4. `last_trope_ids` 最近 5 章用过的不选
5. pulp 模式优先 `cost_weight ≤ 2`，premium 模式可选 `cost_weight = 3`

每章最多 1 个 primary + 1 个 secondary。

# Sub-trope 库

## power-level-up · 升级

- **category**: power
- **subcategory**: 升级
- **market_tier**: sinking
- **cost_weight**: 1
- **best_window**: band_early, after_low, after_humiliation
- **genre_fit**: 玄幻, 都市, 神豪, 末世, 种田, 职场

### 欲望建立

写出主角当前的具体限制：之前做不到 X（具体动作 / 数值 / 场景），让读者感受到这个限制阻碍主角拿到他想要的东西。限制要可量化（能看见三条阵纹 / 月薪三千 / 只能搬一袋米），不能写"实力一般"。

### 阻力加压

让反派或环境利用这个限制压主角：嘲笑、卡资格、抢资源、设局。压力要发生在公开或半公开场合，不能只是主角内心独白。

### 爽点兑现

主角通过前文积累（不能凭空）触发一次具体变化：境界突破 / 技能解锁 / 系统提示 / 工具运转 / 招式成型。变化必须**立刻**完成一次验证——一招破局、一个判断命中、一次产量翻倍、一次权限打开。验证用客观效果说话，不用旁白夸赞。

### 余波钩子

写三层反应：普通人惊讶、懂行的人专业确认、反派或轻视者失态。最后留新问题：更高层的人注意到主角 / 升级带来副作用 / 资源被盯上。

### anti_patterns

- 无铺垫突然升级（必须有前文资源 / 训练 / 危机触发）
- 升级后没有验证动作（只写"他变强了"）
- 一次升级解决主线（只能解决当前小问题）
- 升级到无敌（必须保留更强敌人）
- 旁观者全员无脑震惊（必须分层反应）

### review_signals

- 正文里是否有升级前后的能力差异具体描写？
- 升级是否来自前文积累或当前明确触发？
- 升级后是否立刻完成一次小兑现？
- 是否没有把整条主线打穿？
- 旁观者是否用具体动作 / 表情确认主角变强？

## power-hidden-card · 底牌曝光

- **category**: power
- **subcategory**: 底牌曝光
- **market_tier**: sinking
- **cost_weight**: 2
- **best_window**: band_mid, band_late, after_humiliation, public_scene
- **genre_fit**: 都市, 神豪, 玄幻, 职场, 宫斗

### 欲望建立

写出反派在公开场合把话说死：主角不可能有资格 / 不可能认识某人 / 不可能拿出证据。旁观者接受这个判断，对主角形成公开压力。这一段不能短，要让读者真的相信反派占上风。

### 阻力加压

让旁观者也跟着附和或沉默——主角被孤立。反派加码：限时、当众、有规则后盾。主角尝试低调解决但被逼到必须亮牌。

### 爽点兑现

主角不要急着解释，做一个冷静动作：拨电话 / 递文件 / 拔剑 / 打开权限 / 启动设备。底牌出现时必须有**客观凭证**：印章 / 名单 / 系统记录 / 伤口 / 令牌 / 现场效果。亮牌过程不超过 200 字，越短越冲。

### 余波钩子

写反派的语言断裂、旁观者态度变化、局面控制权转移：保安退开 / 总秘亲自下楼 / 合同重签 / 反派被押走。最后留下：身份曝光带来的更大敌人开始注意主角。

### anti_patterns

- 无伏笔亮底牌（前面必须埋过暗示）
- 主角长篇自夸（让第三方 / 系统 / 权威人物确认）
- 旁观者无理由立刻跪舔（要分层：先沉默、再改口、再讨好）
- 底牌过大导致后续无敌（每次只亮一点，留更高层）
- 频繁套娃"其实还有更大身份"

### review_signals

- 是否有人在前文明确误判主角？
- 亮牌是否发生在公开压力最大时？
- 是否用客观证据证明？
- 反派是否当场失去某个具体东西（职位 / 资格 / 庇护）？
- 旁观者反应是否分层？

## social-face-slap · 打脸

- **category**: social
- **subcategory**: 打脸
- **market_tier**: sinking
- **cost_weight**: 1
- **best_window**: band_early, band_mid, public_scene
- **genre_fit**: 都市, 神豪, 赘婿, 玄幻, 职场, 宫斗, 年代

### 欲望建立

反派或旁观者在公开场合羞辱、质疑、贬低主角——内容必须具体（"你赔不起这十万"、"你连入场资格都没有"），不能写空泛的"看不起"。旁观者起哄附和，主角被孤立。

### 阻力加压

让反派把话说死——给出"如果做不到我就 X"的赌约 / 加码 / 公开发誓。让旁观者觉得反派稳赢。

### 爽点兑现

主角不要嘴炮，拿事实、成绩、身份、证据或结果反击。反击瞬间必须让反派**当场失去面子 / 资格 / 利益 / 话语权**中至少一项。反击动作短、准、狠，不要 ≥ 300 字。

### 余波钩子

旁观者反应分层：有人沉默、有人改口、有人后悔、有人害怕。反派付出具体代价：职位 / 合同 / 资格 / 脸面 / 盟友。最后留：反派背后还有更大势力 / 反派会报复 / 主角被更高层注意。

### anti_patterns

- 只靠主角骂赢（必须有事实证据）
- 反派没有具体损失（必须丢一件实物或权限）
- 旁观者全员震惊无差别（要分层）
- 连续多章重复同一打脸套路（不超过 3 章重复）
- 羞辱铺垫太长、兑现太短（兑现段必须有完整动作链）

### review_signals

- 是否有人明确羞辱主角？
- 是否发生在公共场景？
- 主角反击是否有可验证证据？
- 反派是否当场失去某种东西？
- 旁观者态度是否改变？

## social-status-flip · 地位反转

- **category**: social
- **subcategory**: 地位反转
- **market_tier**: sinking
- **cost_weight**: 2
- **best_window**: band_late, after_humiliation
- **genre_fit**: 都市, 赘婿, 神豪, 宫斗, 职场, 年代

### 欲望建立

写清楚旧地位关系：谁能决定主角去留 / 生死 / 资源 / 名声。让读者看见这个权力差具体怎么压主角——一次具体事件，不是抽象描述。

### 阻力加压

旧地位拥有者再次行使权力压主角：禁止入门 / 拒绝合作 / 公开训斥 / 抢功劳。这次的压制要比之前更狠。

### 爽点兑现

主角通过一件事获得新筹码（合同 / 身份 / 实力 / 上级背书 / 关键证据），然后对方不得不低头。主角**不要急着原谅**，要做选择：原谅 / 拒绝 / 交易 / 清算 / 暂时合作。让读者看见主角掌握选择权。

### 余波钩子

新地位要**落地**：换座位 / 改称呼 / 重签约 / 重新分配资源 / 对方公开退让。最后留：旧地位拥有者会反扑 / 主角的新身份招来更高敌人。

### anti_patterns

- 反转只停留在口头尊重（必须有实物 / 制度变化）
- 对方毫无原因突然低头（必须主角行动造成）
- 主角圣母立刻原谅（保留张力）
- 反转没有后续影响（新地位必须改变下一章行动）
- 反转太频繁导致廉价（≥ 5 章用一次）

### review_signals

- 旧压迫是否明确具体？
- 反转是否由主角行动造成？
- 反转后是否出现具体权力变化（称呼 / 座位 / 合同 / 资源）？
- 对方是否被迫改变姿态？
- 主角是否掌握选择权？

## justice-villain-punished · 恶人受罚

- **category**: justice
- **subcategory**: 恶人受罚
- **market_tier**: sinking
- **cost_weight**: 1
- **best_window**: band_mid, band_late, public_scene
- **genre_fit**: 都市, 玄幻, 末世, 年代, 宫斗, 种田

### 欲望建立

补足反派的具体恶行：抢资源 / 害人 / 欺压弱者 / 诬陷。要让读者**恨得起来**——具体受害者、具体损失、具体场景，不要写"做了很多坏事"。

### 阻力加压

让恶人短暂得意：他以为自己有保护伞 / 证据被处理 / 受害者发不出声。让读者觉得恶人这次又要赢。

### 爽点兑现

主角推动惩罚机制（不是亲自动手，更好）：法律 / 规矩 / 宗门 / 舆论 / 武力 / 因果 / 系统。惩罚落地时写**具体后果**：被押走 / 失去职位 / 被宗门除名 / 财产抄没 / 群众围观唾弃。

### 余波钩子

让恶人知道自己为什么输——给一个对视、一个台词、一个具体细节。让其他想效仿的人感到威慑。最后留：恶人背后的更高势力开始注意主角 / 受害者的下一步如何安置。

### anti_patterns

- 恶人脸谱化但没有具体恶行（必须给至少一个具体受害场景）
- 惩罚只是一句"他完了"（必须有具体处置动作）
- 主角完全旁观（要推动机制启动）
- 恶人受罚后立刻复活继续同样套路（≥ 10 章不复用）
- 过度说教（让动作说话）

### review_signals

- 恶行是否清楚具体？
- 受害者是否具体？
- 主角是否主动推动清算？
- 惩罚是否落地（具体处置动作）？
- 恶人是否失去继续作恶的能力？

## justice-recover-loss · 夺回利益

- **category**: justice
- **subcategory**: 夺回利益
- **market_tier**: sinking
- **cost_weight**: 2
- **best_window**: band_mid, band_late, after_low
- **genre_fit**: 都市, 神豪, 赘婿, 种田, 年代, 末世

### 欲望建立

写清楚被夺走的具体利益：钱 / 地盘 / 功劳 / 资产 / 名誉 / 资源 / 亲情 / 机会。给出原本归属的证据（合同 / 旧物 / 证人 / 记录）。让读者看见利益被抢造成的实际损失。

### 阻力加压

让夺取者享受利益、嘲讽原主：开豪车 / 住主角的房 / 收主角的功劳奖金 / 用主角的名头招摇。这一段越具体越解气。

### 爽点兑现

主角找到证据 / 规则漏洞 / 实力支撑 / 新盟友。夺回时要有**清晰动作**：签字 / 过户 / 收账 / 取回令牌 / 公开名单 / 重新分配。**必须有落袋动作**——不能只写"赢了官司"，要写"账户到账"、"产证更新"、"董事会决议生效"。

### 余波钩子

利益回到主角手中后**立刻产生变化**：欠款还上 / 公司启动 / 反派计划失效 / 主角处境改善。最后留：夺取者背后的人开始反扑 / 资源带来新麻烦。

### anti_patterns

- 利益模糊（必须具体可量化）
- 夺回后没有实际收益（必须改变主角处境）
- 主角只靠别人施舍（要主动争取）
- 证据凭空出现（必须前文有埋）
- 夺取者没有代价（必须失去某个具体东西）

### review_signals

- 被抢利益是否具体可量化？
- 夺回方式是否有合理依据？
- 是否有落袋 / 过户 / 到账动作？
- 夺取者是否失去好处？
- 主角处境是否实际改善？

## mystery-new-clue · 新线索

- **category**: mystery
- **subcategory**: 新线索
- **market_tier**: sinking
- **cost_weight**: 1
- **best_window**: band_early, band_mid
- **genre_fit**: 都市, 玄幻, 末世, 宫斗, 年代

### 欲望建立

提醒读者一个旧问题：之前哪个谜没解 / 谁的身世不清 / 哪个事件无解释。让读者重新意识到这个问题还在。

### 阻力加压

让主角在行动中遇到新阻力：知情人不肯说 / 现场被破坏 / 关键物丢失 / 时间不够。

### 爽点兑现

主角**在行动中**发现一个具体线索：名字 / 符号 / 地址 / 账目 / 伤口 / 气味 / 旧物 / 录音 / 地图缺口。不要凭空旁白告诉读者。主角对线索做一个**初步推断**，但不要全解。

### 余波钩子

章末用线索指向下一步行动：要去哪里 / 要找谁 / 要查什么。线索同时**带出新疑问**——不是全部答案。

### anti_patterns

- 线索过于抽象（必须是具体物件 / 名字 / 地点）
- 线索和主线无关（必须连回当前任务）
- 主角莫名其妙知道答案（必须靠观察 / 推理 / 物证）
- 连续只抛线索不兑现（每 3 章至少回收一个）
- 线索太复杂导致读者忘记（一章一条主要线索）

### review_signals

- 线索是否具体可视？
- 是否和旧问题有关？
- 主角是否主动发现？
- 是否产生下一步行动？
- 是否带出新疑问而非全解？

## emotion-protect · 被保护

- **category**: emotion
- **subcategory**: 被保护
- **market_tier**: sinking
- **cost_weight**: 1
- **best_window**: band_mid, after_humiliation, public_scene
- **genre_fit**: 都市, 赘婿, 萌宝, 宫斗, 年代, 末世

### 欲望建立

写出被保护者面临的真实危险或羞辱：被打 / 被骗 / 被陷害 / 被赶 / 被欺压。危险必须是**当下**的，不是回忆。

### 阻力加压

让旁观者沉默或站在加害者一边，让被保护者孤立无援。主角看到这一幕。

### 爽点兑现

主角不要只说话，用**身体 / 资源 / 身份 / 规则 / 行动**挡住伤害：挡刀 / 认责 / 让座 / 递药 / 公开站队 / 替对方承担后果。动作要短、要狠。被保护者反应要克制但清楚——一个眼神、一句改口、一次第一次主动靠近。

### 余波钩子

写关系变化：称呼变 / 距离变 / 旁观者重新评价这段关系。最后留：加害者会报复 / 被保护者的身世带来新问题 / 主角的保护引来更高敌人。

### anti_patterns

- 只说"我会保护你"（必须有具体动作 / 风险承担）
- 保护没有风险（必须主角付出代价）
- 被保护者毫无反应（必须有可观察的态度变化）
- 强行降智制造保护场景（被保护者不能太蠢）
- 保护后关系没有变化（必须推进一小步）

### review_signals

- 危险是否真实当下？
- 保护是否有具体动作？
- 保护者是否承担代价？
- 被保护者是否感知到变化？
- 关系是否推进一步？

## justice-public-reckoning · 公开清算（贵 / setup-heavy）

- **category**: justice
- **subcategory**: 公开清算
- **market_tier**: mainstream
- **cost_weight**: 3
- **best_window**: band_late, arc_climax, public_scene
- **genre_fit**: 都市, 玄幻, 宫斗, 年代, 末世

> 注：`cost_weight=3` 表示需要前置铺垫章。pulp 模式 selector 默认不选；只在 band 末尾或 arc climax 章手动指定时启用。

### 欲望建立

回收前文：清算对象做过什么、谁是受害者、为什么之前不能动他（庇护伞 / 证据不全 / 时机未到）。把这些前置信息**压缩到 500 字以内**回顾，不要拖。

### 阻力加压

让清算对象在公开场合（家族大会 / 董事会 / 宗门审判 / 法庭 / 直播 / 监察会议）以为自己有保护伞——他刚还在嘲笑主角不敢动。

### 爽点兑现

主角逐条拿证据，证据要**层层升级**：第一份证明他贪 / 第二份证明他害死过人 / 第三份证明他动用过禁权。庇护者态度要**合理转变**——不是突然正义感爆棚，而是利益切割 / 规则压迫 / 证据太硬。最后给出**正式处置**：革职 / 罚没 / 驱逐 / 拘捕 / 宗门除名 / 赔偿。

### 余波钩子

清算对象被押走时，写他和主角的最后一个对视——一句台词暗示"背后还有人"。让其他在场势力开始重新站队。最后留：更高层的反派开始正式注意主角。

### anti_patterns

- 主角空口指控（必须证据链完整）
- 证据一股脑堆出（必须层层升级）
- 清算后没有处罚（必须正式落地）
- 所有人突然正义感爆棚（庇护者转变要有现实理由）
- 反派没有任何反驳（要有挣扎再被打碎）

### review_signals

- 是否公开场景？
- 证据是否逐步升级？
- 清算规则是否明确（法律 / 宗规 / 公司制度）？
- 是否有人从庇护转为切割（带理由）？
- 处置结果是否正式落地？
