## 1. 定位

V4.5 不重新发明一套架构，而是把当前代码、保留设计文档和已落地实现对齐到一个判断口径：

```text
Genesis / Writer / Review 主链
+ Skill Runtime
+ Observability
+ Final BookState Runtime
+ Scheme C BookMap
= V4.5 当前工程基线
```

当前最终方向：

- 书本根真值来自 Genesis。
- 章节生产仍由 `Arc -> Band -> Chapter -> Scene` 主链驱动。
- `BookState + GraphDelta + Snapshot` 是当前 canon 主路径；旧 `world_model_v4` 作为 compatibility projection / migration source。
- 地图最终语义是 `BookMap = SubWorld -> Region -> MapNode -> MapEdge`。
- `SubWorld` 不再表示城市、宗门外门、客栈、遗迹入口等局部舞台。
- 观测、审计、PromptTrace、ArtifactStore 是主链可维护性的底座，不是额外调试代码。

## 3. 当前代码实况

### 3.1 已落地

- Genesis 根层、阶段生成、阶段锁定、`start-writing` handoff 已存在。
- Writer 已支持 scene-aware 生成、fallback、structured output、review/repair/canon 主流程。
- Candidate Draft、Scenario Rehearsal、checkpoint、task-center、governance 和 DecisionEvent 主链已存在。
- Skill Runtime 已有 `forwin/skills/`，并接入 Genesis、Writer、Reviewer 的 prompt layer 与 PromptTrace。
- Observability 已有 `DecisionEvent`、`PromptTrace` 详情、chapter ledger、ArtifactStore、运行事件记录入口。
- Final BookState 已有 protocol、ORM、runtime、repository、projection、compiler、legacy import、review gate 和稳定后端 API response model。
- `WritingOrchestrator` 已调整为 BookState review/compile 优先提交 canon，旧 V4 compiler 失败只记录 compatibility projection failure。
- Scheme C 地图包 `forwin/map/` 已落地：models、protocol、repository、generator、pathfinding、validator、service。
- BookMap 可通过 `create_or_update_book_map()` 生成多个 subworld，并用 `world_gate` 跨 subworld 连通。
- Genesis `start-writing` 已能从 `map_atlas` 自动生成初始 Scheme C BookMap，并把可见地图摘要送入 writer context。
- arc materialization 可从 Genesis atlas 增量补齐缺失 subworld/region，并通过 `world_gate` 接回已有 BookMap。
- Genesis atlas 显式跨 subworld edge 可生成 `InterSubWorldConnectionSpec`；无可解析 edge 时保留默认相邻 `world_gate` fallback。
- `site_state <-> MapNode` bridge 已接入 Genesis anchor / map generation / legacy location import。
- Legacy import 已能把 `SubWorld.metadata_json.region_drafts` 幂等提升为 `map_regions`，并在 migration report 中记录 promoted / skipped / scope conflict。
- Reviewer context 已携带 visible `map_context` 与 reviewer-only objective graph；heuristic reviewer 能检查 objective path、observer-known path、hidden route、blocked route、false route、access rule 和 movement speed policy。
- BookState context assembly 已优先读取 BookState narrative line、knowledge gap、character location overlay，再回退 legacy state repository。
- BookState / map 后端 API 已有稳定 response model：runtime、path、legacy import、map runtime、map path、map ensure-from-genesis。

## 5. V4.5 里程碑

### M1：设计与文档收束

目标：

- 保持当前文档集最小化。
- 任何旧 `SubWorld` 局部语义都只作为 compatibility 出现。
- 新设计入口固定为本文件、`V4_final_book_state_runtime.md`、`map_scheme_c.md`。

验收：

- 旧 V2/V3 历史设计稿不再出现在 `Design-docs/`。
- 活跃文档不再把城市、宗门、遗迹入口当作 `SubWorld`。

### M2：BookState 成为主 canon runtime

状态：后端主链已补齐；Writer/extractor 直接产出完整 `GraphDelta` 仍是后续优化。

目标：

- Writer/extractor 直接产出或稳定转换为 `ApprovedGraphDeltaSet`。
- BookState compiler 成为 canon commit 的主路径。
- 旧 V4 world model 降级为 compatibility projection / migration source。

验收：

- Accepted chapter 后能从 BookState snapshot/replay 重建 world/map/cognition runtime。
- Context assembly 优先读取 BookState。
- 旧 `entities / entity_states / relation_edges` 只作为兼容投影或迁移来源。

### M3：Scheme C 地图接入主链

状态：初始生成、arc expansion、`world_gate` 接入、site_state bridge 和最小 map API 已补齐。

目标：

- Genesis map seed 可生成 `SubWorldMapSpec`。
- `start-writing` 或 arc initialization 能创建初始 BookMap。
- arc expansion 能创建新 subworld 或 region，并通过 `world_gate` / portal / route 接入 BookMap。

验收：

- 新项目可自动生成至少一个 subworld map。
- writer context 可拿到主角当前位置、region、nearby nodes、reachable nodes。
- reviewer 可用地图 travel time 检查章节移动。

### M4：Cognition-aware reviewer

状态：objective path / observer-known path 分离、hidden route、blocked route、false route、known detour deterministic issue code 已补齐。

目标：

- reviewer 区分 objective path 和 observer-known path。
- hidden route 未发现时不能被角色直接使用。
- false edges 可导致错误认知路径，但不能污染 objective map。

验收：

- 同一移动可同时给出 objective reachable 与 known unreachable / detour。
- hidden route、blocked route、false route 均有 deterministic issue code。

### M5：Observability hardening

状态：BookState review/compile、map generation/expansion、map movement review issue、legacy projection failure 已接入 `DecisionEvent`。

目标：

- BookState compile、map generation、pathfinding review、cognition override 都写入可追踪 ledger。
- PromptTrace / DecisionEvent / ArtifactStore 形成统一调试路径。

验收：

- 单章失败能从 task timeline 定位到 review、map、BookState 或 LLM 层。
- BookState / map debug API 返回稳定 contract。

### M6：Migration and cleanup

状态：legacy import 已返回 migration report，并为 legacy location/city/ruin 生成 `site_state <-> MapNode` binding；不会把城市/宗门/遗迹误建为 `SubWorld`。

目标：

- legacy SubWorld region drafts 提升为 `map_regions`。
- 旧 world_model_v4 rows 可导入 BookState。
- 旧局部 SubWorld 项目能给出迁移报告。

验收：

- 迁移不会把城市/宗门/遗迹误建为 subworld。
- validation report 能列出需要人工处理的 legacy scope conflict。

## 6. 当前非目标

V4.5 不做：

- 不做真实地理板块模拟。
- 不做 tile renderer。
- 不把 Neo4j 作为主存储。
- 不把地图路线塞进普通 `WorldEdge`。
- 不让 Writer 直接看到 hidden objective truth。
- 不把 Skill Runtime 扩成脚本执行平台。
- 不把 debug API 宣称为外部稳定 API。

## 7. 当前风险

- BookState 与旧 V4 compiler 仍双轨并存，但旧 V4 已降级为 compatibility projection；主要风险是投影延迟或调试噪声。
- `V2_9_2.md` 仍有历史兼容语义，开发时必须以 V4.5 / Scheme C 覆盖解释。
- V4.5.1 只继续收束后端 contract、migration audit、trace payload、movement policy 文档；复杂多层 route policy、剧情时间窗口、交通工具体系和完整规则包转入 V4.6+。
