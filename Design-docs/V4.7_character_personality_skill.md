# ForWin V4.7 Character Personality Skill 设计文档

更新时间：2026-04-27

状态：设计落档；首轮 runtime、BookState loadout、World Studio 编辑入口、writer/reviewer 接入和 v2 skill 仓库骨架已建立。后续重点是补全 41 个 skill 内容、扩展 reviewer rule pack、把读者 OOC 反馈映射回 skill 维护流程。

## 1. 核心定位

ForWin V4.7 引入 `Character Personality Skill Library`，用于解决长篇连载中人物“扁平、跨章节漂移、压力反应随机、不同关系对象表现一致”的问题。

它不是：

- 作者人格
- 写作 agent 的工作风格
- 单个角色卡
- 临床诊断标签
- MBTI / 九型 / DISC 等模型的一对一转换表
- 固定剧情结论

它是：

```text
可复用的人物行为机制：
这种人如何感知、判断、说话、行动、亲近、疏离、失控、恢复、成长、堕落。
```

V4.7 的位置：

```text
V4.5 BookState canon runtime
  ↓
V4.6 Knowledge System projection / LLM KB
  ↓
V4.7 Character Personality Skill runtime
  ↓
writer / reviewer / World Studio
```

V4.7 不改变 canon 权威边界。人物性格 skill 只能解释和约束表现倾向，不能改写角色已经做过什么、知道什么、承诺过什么。

## 2. 设计目标

目标：

- 角色可以挂多个 personality skill。
- 同一角色跨章节表现稳定但不僵硬。
- 同一 skill 在不同角色上因角色卡、目标、关系、权重、压力触发不同而表现不同。
- writer prompt 只接收压缩后的 `active_personality_context`，不塞入完整 skill 仓库。
- reviewer 可以检查 OOC、倾向绝对化、社交面具变成本体、stress mode 常驻、reference model 越权等问题。
- World Studio 能编辑角色 `personality_loadout`。
- 后续读者反馈中的“人物 OOC”可以定位到具体 skill、loadout、relationship state 或 canon 冲突。

非目标：

- 不替代 Character Card。
- 不存储角色 canon。
- 不让 personality skill 覆盖剧情状态或 scene goal。
- 不把心理学模型当运行时行为引擎。
- 不强制所有挂同一 skill 的角色说话完全一样。

## 3. 数据边界

### 3.1 Character Card

角色卡回答：

```text
这个人是谁？
```

包含姓名、身份、阵营、出身、当前目标、关系网、已发生 canon、秘密、能力、剧情状态。

### 3.2 Character Personality Skill

personality skill 回答：

```text
这种人格机制通常如何运作？
```

它只能写通用机制：

- 感知偏差
- 价值排序
- 决策触发器
- 关系对象差异
- 对白、动作、沉默、撒谎、示好方式
- 轻压、中压、极压下的变形
- 健康成长和负向弧线

禁止写入 reusable skill 的运行时字段：

```yaml
character_id: 禁止
character_name: 禁止
current_goal: 禁止
canon_state: 禁止
scene_context: 禁止
current_relationship_score: 禁止
current_relationship_state: 禁止
current_arc_position: 禁止
current_chapter_goal: 禁止
active_personality_context: 禁止
```

### 3.3 Personality Loadout

角色卡只挂 loadout，不展开完整 skill 内容。

```yaml
personality_loadout:
  dominant:
    skill: trait-suspicious-survivor
    weight: 0.75

  secondary:
    - skill: trait-loyal-protector
      weight: 0.55

  social_mask:
    - skill: mask-cold-professional
      weight: 0.60
      active_when:
        - public_scene
        - superior_present

  stress_modes:
    - skill: stress-paranoid-controller
      trigger:
        - betrayal_suspected
        - information_control_lost

  relationship_patterns:
    - target: char_rival
      skill: rel-rival-respect
      weight: 0.50

  overrides:
    trait-suspicious-survivor:
      trust_baseline: low
```

当前落点：`WorldNode.profile.personality_loadout`。

## 4. Skill 类型

### 4.1 trait

基础性格机制，最重，使用完整六层模型。

回答：

```text
这个人平时如何感知世界、排序价值、做决定、处理关系、表达情绪、在压力下变形。
```

### 4.2 social_mask

人物对外展示的社交面具。它不是本体。

回答：

```text
别人看到什么？
它遮住什么？
什么场景启动？
什么压力会让它裂开？
裂开时露出什么？
```

social mask 不能覆盖 dominant trait，只能改变外显呈现。

### 4.3 stress_mode

压力下被激活的短期变形模式。

回答：

```text
什么触发它？
它如何放大原性格？
轻压、中压、极压有什么区别？
如何恢复？
它对关系造成什么成本？
```

stress mode 不能常驻。

### 4.4 relationship_pattern

关系中的稳定模式。

回答：

```text
这个人在亲密、信任、边界、冲突、和解中如何运作。
```

relationship pattern 不是实时关系分值。实时 trust / resentment / debt / attraction 来自 `relationship_state`。

### 4.5 archetype

叙事功能原型。

回答：

```text
这个角色在故事结构中承担什么功能？
它照亮什么主题？
它推动什么冲突？
```

archetype 不能单独决定对白和行动。

## 5. 六层人物性格模型

每个 trait 使用完整六层：

```text
Layer 1: 基础人格轴
Layer 2: 核心驱动力
Layer 3: 决策机制
Layer 4: 关系模式
Layer 5: 表达与社交面具
Layer 6: 压力变形与成长弧线
```

Layer 5 必须包含：

- dialogue behavior
- body language
- affection style
- anger style
- lie style
- silence style
- humor style

原因：小说人物的稳定性不只来自对白，也来自动作、停顿、站位、触碰反应、沉默方式。

## 6. Reference Model Policy

`reference_models` 是解释坐标，不是行为引擎。

运行时优先级：

```text
trigger_rules
> relationship_rules
> expression
> stress_and_arc
> do_not
> prompt_compression
> reference_models
```

硬规则：

```yaml
reference_model_policy:
  - Reference models are explanatory anchors only.
  - Runtime generation must follow behavior rules first.
  - Do not infer new behavior from MBTI, Enneagram, DISC, Socionics, or archetype labels directly.
  - If a reference model conflicts with trigger_rules or relationship_rules, ignore the reference model.
  - Do not use clinical diagnosis labels as primary skill names.
```

## 8. Runtime Loading

加载步骤：

```text
1. 读取角色卡。
2. 读取角色当前 canon 状态。
3. 读取当前 scene goal。
4. 读取当前 relationship state。
5. 加载 dominant personality skill。
6. 加载达到阈值的 secondary skill。
7. 场景条件匹配时加载 social mask。
8. trigger 匹配时加载 stress mode。
9. target 匹配时加载 relationship pattern。
10. 合并为 active_personality_context。
```

合并优先级：

```text
canon facts
> current plot state
> scene goal
> relationship state
> active stress mode
> dominant personality skill
> secondary personality skill
> social mask
> prose style
```

输出格式：

```yaml
active_personality_context:
  character_id: char_example
  active_skills:
    dominant:
      - trait-suspicious-survivor
    secondary:
      - trait-loyal-protector
    social_mask:
      - mask-cold-professional
    stress_mode:
      - stress-paranoid-controller
  current_behavior_bias:
    perception:
      - 先看谁隐瞒、谁受益、哪里能撤
    decision:
      - 先保退路，再承诺
    dialogue:
      - 短句、反问、要求证据
    body_language:
      - 扫出口，不背对陌生人
    relationship_behavior:
      - 对陌生人低披露，对盟友条件信任
    stress_behavior:
      - 信息缺口会激活控制信息流
  constraints:
    - Do not override canon.
    - Do not make the character omniscient.
    - Do not flatten the character into one repeated behavior.
    - Do not infer behavior from model labels.
```

## 9. Writer 接入

Writer prompt 中新增 `【人物性格运行时】` 段落，只使用 active context 的压缩 bias。

写作要求：

- 先看当前目标和关系对象。
- dominant skill 作为第一反应。
- secondary skill 作为修正、迟疑、矛盾或反差。
- social mask 只控制外显。
- stress mode 只在 trigger 激活时生效。
- canon、scene goal、relationship state 优先。

## 10. Reviewer 接入

Reviewer payload 将 active personality context 作为 evidence：

```text
personality:{character_id}
```

检查项：

1. 决策是否符合 active dominant skill。
2. 偏离是否有 canon、relationship、stress 或 growth 理由。
3. 对白是否符合 expression layer。
4. 动作是否符合 body language layer。
5. stress response 是否匹配压力等级。
6. 是否区分不同关系对象。
7. 是否把倾向写成绝对规则。
8. 是否把角色写成重复口癖。
9. reference model label 是否越权。
10. skill 是否发明新 canon。
11. social mask 是否变成角色本体。
12. stress mode 是否无 trigger 常驻。
13. 成长是否删除了人格机制。

## 11. World Studio 接入

World Studio 新增人物性格编辑入口：

- 读取 personality skill catalog。
- 展示 BookState character 列表。
- 编辑 `personality_loadout` JSON。
- 快速选择 dominant trait。
- 保存后写入 character `WorldNode.profile.personality_loadout`。
- 通过 DecisionEvent 记录 `personality_loadout_updated`。

## 12. API 边界

当前 API：

```text
GET /api/personality-skills
GET /api/projects/{project_id}/book-state/characters/personality
GET /api/projects/{project_id}/book-state/characters/{character_id}/personality-loadout
PUT /api/projects/{project_id}/book-state/characters/{character_id}/personality-loadout
```

写入规则：

- API 校验 skill id 是否存在。
- API 只写 character profile 中的 loadout。
- API 不修改 skill 文件。
- API 不改写 canon 行为事实。

## 13. 冲突处理规则

Dominant vs Secondary：

```text
dominant 决定第一反应。
secondary 决定修正、迟疑、矛盾、反差。
```

Social Mask vs Dominant：

```text
social_mask 负责别人看到什么。
dominant 负责人物真正如何处理信息。
```

Stress Mode vs Social Mask：

```text
stress_mode 可以击穿 mask。
```

Skill vs Canon：

```text
canon 永远优先。
personality skill 只能解释 canon，不能改写 canon。
```

## 15. 版本验收标准

V4.7 完整验收需要满足：

- 41 个 skill 不只是占位，至少 trait/social_mask/stress_mode 的核心字段完整。
- 每个 skill 有 body language 和 prompt compression。
- reference model 不参与运行时优先级。
- writer 不加载完整 skill 仓库。
- reviewer 能指出 OOC、绝对化、模型标签越权、stress mode 常驻。
- World Studio 可以稳定编辑 loadout。
- 角色跨章节一致性测试可通过。
- 同 skill 不同角色差异测试可通过。
- canon 优先级不可被 personality skill 覆盖。
