## 1. 系统定义

ForWin 是面向长篇连载网文的自动化写作系统。

核心链路：

```text
Arc -> Band -> Chapter -> Scene
Planning -> Provisional -> Writer -> Review -> Repair -> Canon -> Post-acceptance -> Governance/Feedback
```

支持三种运行模式：

| 模式 | 语义 |
| --- | --- |
| `blackbox` | 自动推进；遇到治理阻断、checkpoint warn/fail/error、运行异常时暂停。 |
| `copilot` | 自动写作和修复；关键失败后进入人工 review。 |
| `checkpoint` | 产出后暂停；等待人工审查和批准。 |

## 2. 分层架构

| 层 | 来源 | 职责 | 主对象 |
| --- | --- | --- | --- |
| 生产主链 | V2.3 | 写作、结构化抽取、review、repair、canon | `ChapterPlan`、`WriterOutput`、`ChapterReview` |
| 反馈校准层 | V2.6 | 评论信号、聚合、趋势、动作建议 | `CommentSignalCandidate`、`SignalWindowAggregate`、`AudienceHintPack` |
| 体验审查层 | V2.7 | 网文体验计划、WNER、lint、repair patch | `ReaderPromise`、`BandDelightSchedule`、`ReviewContextPack` |
| 治理层 | V2.8 | 严格推进、checkpoint、future constraints、审计 | `BandCheckpoint`、`NarrativeConstraint`、`DecisionEvent` |

---

## 3. 规划模型

| 层级 | 职责 |
| --- | --- |
| `Project` | 一本书的根对象。 |
| `Arc` | 长线剧情阶段。 |
| `Band` | 3-5 章左右的治理与执行单元。 |
| `Chapter` | 正式连载章节。 |
| `Scene` | Writer 内部生成单元。 |

Band 是 V2.8.1 的一等治理单元，聚合 band 计划、task contract、provisional、checkpoint、future constraints、next-band compatibility 和 decision events。

---

## 4. Writer 主链

默认流程：

```text
Scene Breakdown -> Scene Generation -> Scene Stitch -> Structured Extraction
```

保留 single-chapter generation 作为 fallback。

`WriterOutput` 必须包含：

- 正文：`chapter_number`、`title`、`body`、`char_count`、`end_of_chapter_summary`
- scene 结果：`scene_outputs`
- 结构化抽取：`state_changes`、`new_events`、`thread_beats`、`time_advance`
- 运行信息：`generation_meta`

错误恢复：

- 单模型 retry。
- 跨 profile fallback。
- parse repair 或 fallback。
- blackbox 多次 repair 失败后可 forced accept 当前最新版 draft。
- forced accept 仍必须经过 band checkpoint。

---

## 5. Review 与 Repair

Review 聚合：

- continuity review
- WNER
- lint signals

Verdict：

| verdict | 语义 |
| --- | --- |
| `pass` | 允许进入 canon。 |
| `warn` | 可继续，但必须记录 issue。 |
| `fail` | 触发 repair 或人工处理。 |

Repair scope：

```text
scene -> band -> arc
```

运行模式差异：

| 模式 | fail 后行为 |
| --- | --- |
| `checkpoint` | 不自动 rewrite。 |
| `copilot` | 自动修复；耗尽后 `needs_review`。 |
| `blackbox` | 自动修复；耗尽后可 forced accept，但 checkpoint 仍可暂停。 |

---

## 6. Canon 与 Post-acceptance

只有 accepted chapter 可以写入 canon。

Canon 回写：

- entities
- events
- plot threads
- timeline
- world state snapshots

Post-acceptance：

- memory index upsert
- pacing assessment
- stage analysis
- world pressure update
- NPC intent snapshot
- feedback aggregation

世界状态采用追加式记录，不使用单个大 JSON 覆盖真值。

---

## 7. 反馈校准层

支持六类 signal：

- `risk`
- `pacing`
- `confusion`
- `character_heat`
- `relationship_interest`
- `prediction`

数据流：

```text
PublisherRawComment
-> CommentSignalCandidate
-> SignalWindowAggregate
-> ReaderScaleSnapshot
-> score_v1
-> AudienceTrendView
-> ActionMapper
-> AudienceHintPack
```

边界：

- audience feedback 不直接修改 canon。
- audience feedback 只作为 planning、pacing、experience、writer hint 的校准信号。

---

## 8. 体验审查层

Experience overlay：

- `ReaderPromise`
- `ArcPayoffMap`
- `BandDelightSchedule`
- `ChapterExperiencePlan`

WNER 使用 `ReviewContextPack`，包含：

- chapter/band/arc overlay
- entity snapshots
- rule snippets
- recent canon events
- recent review notes
- timeline
- world pressure
- audience hints
- lint signals

WNER 输出：

- planned reward tags
- delivered reward tags
- 7 维 experience scores
- evidence-anchored issues
- review notes
- repair instruction

7 维 scores：

- `narrative_understanding`
- `attentional_focus`
- `emotional_engagement`
- `narrative_presence`
- `payoff_delivery`
- `stall_tolerance`
- `hook_efficiency`

Trope registry 使用 JSON seed。V2.8.1 不要求内置完整运营级 trope 库。

## 9. 治理层

### 9.1 默认策略

新项目和新 blackbox task 默认：

- `progression_mode = serial_canon_band_guard`
- `auto_band_checkpoint = true`
- `manual_checkpoints_enabled = true`
- `future_constraints_enabled = true`

旧项目保持 legacy 兼容。

### 9.2 推进策略

| 策略 | 语义 |
| --- | --- |
| `legacy_relaxed` | 兼容旧行为。 |
| `serial_canon` | 前一章 accepted 后才能写后一章。 |
| `serial_canon_band_guard` | 跨 band 时要求上一 band checkpoint 为 `pass` 或 `overridden`。 |

### 9.3 Band Checkpoint

固定检查：

- band 内章节全部 accepted
- 未处理 chapter review
- latest review fail
- provisional fail
- band task contract
- next-band compatibility
- active future constraints
- future resource preservation

Status：

| status | 行为 |
| --- | --- |
| `pass` | 自动继续。 |
| `warn` | 默认暂停。 |
| `fail` | 暂停，进入 repair 或人工处理。 |
| `error` | 暂停，记录 evaluator error。 |
| `overridden` | 人工放行。 |

Auto checkpoint 必须按 `project / band / boundary_kind / boundary_chapter` 幂等。

### 9.4 Manual Checkpoint

必须包含：

- `boundary_kind`
- `boundary_chapter`
- `reason`

允许 boundary：

- `chapter_start`
- `chapter_accepted`
- `band_end`

### 9.5 Task Contract

使用 JSON contract，不新增任务表。

Task 类型：

- 必须推进
- 必须建立
- 必须埋伏
- 必须保留
- 必须交付体验

Chapter review 检查 chapter contract。Band checkpoint 检查 band contract。修改必须写 `DecisionEvent`。

### 9.6 Future Constraints

使用 `NarrativeConstraint`。

| level | 语义 |
| --- | --- |
| `hard` | 阻断。 |
| `soft` | warning。 |
| `hint` | info。 |

`protect_until_chapter` 表示保护到该章节结束。

关闭 `future_constraints_enabled` 后，constraint 可保存和展示，但不进入 context、review、checkpoint 或 blocking reason。

### 9.7 Issue Group

Review/checkpoint issue 必须归入：

- `fact_conflict`
- `director_imbalance`
- `runtime_observation`
- `governance_action`

## 10. 决策日志

`DecisionEvent` 是唯一结构化日志真值。

必须记录：

- generation / continue / run start / run completed
- hard gate hit
- review verdict
- repair chain
- forced accept
- canon commit/fail
- checkpoint create/hit/approve/override
- constraint lifecycle
- governance update
- pause/continue/terminate
- LLM retry/fallback

所有人工动作必须带 reason。

Causal replay 基于：

- `causal_root_id`
- `parent_event_id`
- `related_object_type`
- `related_object_id`

---

## 11. 前端入口

必须提供：

- 项目级治理设置
- 运行级 governance override
- chapter/band manual checkpoint
- task contract 编辑
- constraint 编辑
- blocking reason 决策解释
- checkpoint / review / decision event 跳转

Blocking reason 必须展示：

- gate code
- chapter/band
- latest checkpoint
- checkpoint issues
- review issue groups
- decision event link

---

## 12. 稳定接口

核心对象：

- `ProjectGovernanceSettings`
- `PlanTaskItem`
- `NarrativeConstraintInfo`
- `BandCheckpointDetail`
- `DecisionEventInfo`
- `GenerationControlInfo.blocking_reason`
- `GenerationControlInfo.next_gate`

核心 API：

- task contract read/write
- constraint create/update/archive
- manual checkpoint create
- checkpoint approve/override
- review approve
- decision events
- causal replay
- governance insights
- trope templates
- band experience override

---

## 13. V2.8.1 不包含

- 自动理解所有隐性未来需求。
- 完整 188 trope 模板库。
- 独立任务表。
- 第二套日志真值。
- 第二套约束推理引擎。
- 完整 scene continuation contract。
- 生产级 reader estimate 精准校准。
- 完全无 heuristic 的 WNER。
