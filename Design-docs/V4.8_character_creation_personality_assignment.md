# ForWin V4.8 Character Creation Helper 与自动性格分配设计文档

更新时间：2026-04-28

状态：目标态与四阶段更新计划落档。本文档用于指导 ForWin 从 V4.7 的“人物性格 runtime 与手工 loadout 编辑”升级为 V4.8 的“统一人物创建契约与自动 personality assignment”。

## 1. 核心定位

V4.8 要把“创建人物”从分散在 API、SubWorld、legacy import、GraphDelta、World Studio 和 orchestration 里的临时动作，收口为一个统一领域能力：

```text
创建一个命名人物
= 建立 canonical character identity
+ 创建或更新 BookState WorldNode(node_type="character")
+ 兼容 legacy Entity 投影
+ 建立必要 roster / relation / location 绑定
+ 自动分配 personality_loadout
+ 写入 personality_assignment report
+ 通过 character integrity gate
+ 写入 DecisionEvent 审计
+ 让 writer/reviewer 从第一次使用起读取 active_personality_context
```

V4.8 不改变 V4.7 的 runtime 权威落点：

```text
WorldNode.profile.personality_loadout
  -> build_active_personality_contexts
  -> writer prompt: 人物性格运行时
  -> reviewer evidence: personality:{character_id}
```

V4.8 改变的是创建契约：最终系统不允许出现“已经是命名 character 但没有有效 personality_loadout”的裸角色。

## 2. 当前代码契合度

当前代码已经具备以下基础：

- `WorldNode.profile.personality_loadout` 已是 BookState character profile 的合法字段。
- `forwin/personality` 已有 `PersonalityLoadout`、`CharacterPersonalityLibrary` 和 active context 构建逻辑。
- context assembler 已从 BookState character 读取 loadout 并注入 `active_personality_contexts`。
- writer prompt 已注入压缩后的“人物性格运行时”，不直接塞完整 skill 仓库。
- reviewer payload 已把 personality context 编入 `personality:{character_id}` evidence。
- World Studio 已有基础 personality loadout JSON 编辑器。

主要缺口：

- 没有 `forwin/characters` 领域包。
- 没有 `CharacterCreationHelper`。
- 没有 `PersonalityLoadoutAssigner`、assignment policy、assignment report 与 catalog-driven scoring。
- SubWorld、planned slot、legacy import、GraphDelta、旧 orchestrator seed 仍存在裸建 character 路径。
- `PersonalityLoadout` 当前偏宽松：dominant 可空，stress trigger/social mask active_when 没有严格 gate。
- 只有手工更新 loadout 的 DecisionEvent，没有自动分配、重分配、backfill、integrity failure 的审计事件。
- coverage/backfill API 不存在。

因此 V4.8 是破坏性更新，必须按阶段推进。

## 4. Canonical 数据边界

### 4.1 BookState 是 runtime 权威

运行时只以 BookState character node 为准：

```text
WorldNode.profile.personality_loadout
WorldNode.metadata.personality_assignment
WorldNode.metadata.character_creation
```

legacy `Entity(kind="character")` 只作为兼容投影，不保存完整 canonical loadout。

### 4.2 Roster 是作用域绑定

`SubWorldRosterItem` 表示角色属于哪个 subworld / slot / band / role，不拥有 personality runtime 权威。

最终绑定规则：

```text
SubWorldRosterItem.entity_id -> legacy Entity
SubWorldRosterItem.metadata.character_id -> BookState WorldNode character_id
```

### 4.3 Assignment 不写 canon

personality assignment 只能生成行为倾向机制，不允许生成角色事实、秘密、知识、经历、关系结论或剧情动作。

## 5. 四阶段更新计划

### Phase 1: Personality Assignment Foundation

目标：建立自动分配的独立领域层，但不立刻改造所有创建入口。

新增：

- `forwin/personality/assignment.py`
- `forwin/personality/policy.py`
- `forwin/personality/validation.py`
- assignment report / validation report / policy models
- assignment catalog YAML
- `personality_assignment.schema.json`

实现：

- `PersonalityLoadoutAssigner.assign()`
- `PersonalityLoadoutAssigner.preview()`
- `PersonalityLoadoutAssigner.validate()`
- explicit / existing / manual loadout 优先级
- deterministic rule scoring
- fallback minimal loadout
- unknown skill id fallback 或 block
- dominant skill type 校验
- stress trigger 校验
- social mask 不能作为命名人物唯一机制
- assignment report 输出 confidence、status、reason_tags、selected/candidate/rejected skills

本阶段不做：

- 不强制所有旧角色都有 loadout。
- 不改造所有裸建入口。
- 不让 repository 依赖 assignment policy。

关键测试：

```bash
uv run --extra test python -m pytest tests/test_personality_assignment.py -q
uv run --extra test python -m pytest tests/test_personality_runtime.py -q
```

验收：

- 相同输入、相同 policy、相同 catalog 下 assignment 结果稳定。
- protector/professional golden case 可自动得到 `trait-loyal-protector + mask-cold-professional`。
- low confidence 可进入 fallback，并标记 `valid_needs_review`。
- manual/existing loadout 默认被 preserve。

### Phase 2: CharacterCreationHelper And Non-Breaking APIs

目标：引入统一人物创建 helper，先服务新 API 和可控路径，不立即禁用旧入口。

新增：

- `forwin/characters/models.py`
- `forwin/characters/registry.py`
- `forwin/characters/integrity.py`
- `forwin/characters/creation.py`
- `forwin/characters/events.py`

核心接口：

```python
class CharacterCreationHelper:
    def create_character(self, request: CharacterCreationRequest) -> CharacterCreationResult: ...
    def get_or_create_character(self, request: CharacterCreationRequest) -> CharacterCreationResult: ...
    def materialize_roster_character(self, request: RosterMaterializationRequest) -> CharacterCreationResult: ...
    def import_legacy_character(self, request: LegacyCharacterImportRequest) -> CharacterCreationResult: ...
    def apply_book_state_character_patch(self, request: BookStateCharacterPatchRequest) -> CharacterCreationResult: ...
    def ensure_character_integrity(self, character_id: str, *, reason: str) -> CharacterIntegrityResult: ...
```

新增 API：

```text
POST /api/projects/{project_id}/characters
POST /api/projects/{project_id}/characters/personality/preview
GET  /api/projects/{project_id}/characters/personality/coverage
POST /api/projects/{project_id}/characters/personality/backfill
POST /api/projects/{project_id}/characters/{character_id}/personality/reassign
GET  /api/projects/{project_id}/characters/{character_id}/personality/assignment-report
```

兼容策略：

- 保留现有 `/book-state/characters/{id}/personality-loadout`。
- World Studio 手工保存 loadout 时写 `manual_override=true`。
- Backfill 默认不覆盖 manual override。
- strict gate 暂不全局启用。

关键测试：

```bash
uv run --extra test python -m pytest tests/test_character_creation_helper.py tests/test_personality_assignment.py -q
uv run --extra test python -m pytest tests/test_world_studio_frontend.py -q
```

验收：

- 新 API 创建命名人物后，BookState character profile 有有效 `personality_loadout`。
- response 返回 assignment report。
- manual override 被 preserve。
- coverage API 能统计 valid/missing/fallback/manual/needs_review。

### Phase 3: Creation Entrypoint Migration

目标：把当前分散裸建 character 的路径迁移到 `CharacterCreationHelper`。

改造入口：

- SubWorld `core_named_characters`。
- SubWorld planned slot materialization。
- legacy Entity import。
- BookState GraphDelta `NodePatch(op="create", node_type="character")`。
- arc plan seed / legacy orchestrator character seed。

规则：

- `StateUpdater.create_entity(kind="character")` 不立即删除，但业务代码不得新增裸调用。
- GraphDelta 创建 character 时，落库前必须补齐 loadout 或阻断。
- legacy import 不直接创建裸 `WorldNode(node_type="character")`。
- roster materialization 必须写 `metadata.character_id`。

新增治理测试：

```text

```

允许例外：

- `CharacterCreationHelper` 内部。
- 测试构造器。
- schema fixture。
- migration dry-run fixture。

关键测试：

```bash

uv run --extra test python -m pytest tests/test_subworld_control.py tests/test_book_state_legacy_import.py -q
uv run --extra test python -m pytest tests/test_book_state_repository_projection_compiler.py -q
```

验收：

- SubWorld core named character 自动拥有 loadout。
- planned slot materialized character 自动拥有 loadout。
- legacy imported character 自动拥有 loadout。
- GraphDelta create character 不再能生成裸 BookState character。

### Phase 4: Strict Integrity Gate, Runtime Enforcement, UX Completion

目标：启用最终态约束，让裸命名角色无法进入 writer/reviewer runtime。

Runtime enforcement：

- context assembler 构建 chapter context 前执行 character integrity gate。
- strict policy 下 missing loadout 阻断生成。
- non-strict policy 下记录 integrity warning，并可触发 repair/backfill。

Reviewer enforcement：

新增 issue taxonomy：

```text
personality_missing_loadout
personality_assignment_low_confidence
personality_ooc
stress_mode_without_trigger
social_mask_as_self
loadout_skill_missing
loadout_skill_type_mismatch
relationship_pattern_missing_target
```

World Studio completion：

- 人物创建表单。
- 自动性格 preview。
- assignment report view。
- active context preview。
- manual override lock。
- coverage dashboard filters。

Observability：

- `personality_assigned` structured log。
- coverage ratio。
- fallback used count。
- manual override count。
- low confidence count。
- missing loadout integrity count。

关键测试：

```bash

uv run --extra test python -m pytest tests/test_book_state_runtime.py tests/test_book_state_repository_projection_compiler.py tests/test_world_v4_runtime_loop.py -q
```

验收：

- 任意入口创建命名人物后，BookState character profile 必有有效 `personality_loadout`。
- writer prompt 只包含 compressed active personality context。
- reviewer payload 能引用 `personality:{character_id}`。
- manual override 不被 backfill 或默认 reassign 覆盖。
- strict gate 能阻断裸命名角色。
- coverage dashboard 能筛选 missing/fallback/needs_review/manual/stress/social mask 问题。

## 6. Integrity Gate 规则

阻断级别：

```text
error:
  named character missing personality_loadout
  dominant skill missing for core/supporting character
  loadout references unknown skill id
  dominant references non-trait skill
  stress mode without trigger
  social mask as only mechanism for named character
  manual override loadout invalid schema

warning:
  fallback used
  low confidence assignment
  secondary too many
  social mask missing active_when but policy allows default public mask
  relationship pattern missing target but skill allows global pattern
```

Gate 时机：

```text
before writer context assembly
before BookState snapshot materialization
before chapter acceptance
before project export
before World Studio save
```

## 7. DecisionEvent 审计

V4.8 新增或使用以下事件类型：

```text
CHARACTER_CREATED
CHARACTER_MERGED_EXISTING
CHARACTER_ROSTER_MATERIALIZED
CHARACTER_IMPORTED_FROM_LEGACY
PERSONALITY_LOADOUT_AUTO_ASSIGNED
PERSONALITY_LOADOUT_REASSIGNED
PERSONALITY_LOADOUT_MANUAL_OVERRIDE
PERSONALITY_ASSIGNMENT_BACKFILL_COMPLETED
PERSONALITY_RELATIONSHIP_ENRICHED
CHARACTER_INTEGRITY_CHECK_FAILED
```

每次创建、自动分配、重分配、manual override、backfill、integrity failure 都必须有 DecisionEvent。

## 8. API 与 UI 最终态

## 10. 最终不变量

```text
Invariant 1:
  Every named BookState character must have a valid personality_loadout.

Invariant 2:
  Automatic assignment never overwrites manual override by default.

Invariant 3:
  Repository persists character data; it does not decide personality policy.

Invariant 4:
  CharacterCreationHelper is the only sanctioned boundary for character creation.

Invariant 5:
  PersonalityLoadoutAssigner is deterministic and explainable.

Invariant 6:
  Personality skill constrains behavior tendency, never canon facts.

Invariant 7:
  Writer receives active_personality_context, never full skill repository.

Invariant 8:
  Reviewer can trace personality behavior to evidence id personality:{character_id}.

Invariant 9:
  stress_mode requires trigger.

Invariant 10:
  social_mask cannot be the character's entire personality.
```

## 11. 默认假设

- BookState character node 是 personality runtime canonical source。
- legacy Entity 保留，但只作为兼容投影。
- Phase 1-2 不强制旧角色全部有 loadout。
- Phase 3 完成入口迁移后，Phase 4 再启用 strict gate。
- `StateUpdater.create_entity(kind="character")` 先限制新增业务调用，不立即删除。
- manual override 默认永不被 backfill 或 reassign 覆盖。
- fallback skill 必须真实存在；若目标 fallback skill 缺失，Phase 1 必须先补 skill 或改用现有真实 trait。

## 12. Final Gap Closure 增量实现状态

Final Gap Closure 增量在已有 V4.8 核心闭环上继续补齐以下能力：

- `CharacterCreationHelper` 作为人物创建的统一入口，负责 BookState character、legacy Entity 投影、初始 state snapshot、personality assignment 与审计事件。
- `PersonalityLoadoutAssigner` 支持 catalog-driven scoring、fallback、manual/explicit preserve、schema-level validation、assignment report、secondary trait 组合、compatibility/tension warning、hard conflict reject、cast diversity adjustment 和 seeded deterministic tie-break。
- `CharacterPersonalityPolicyResolver` 从 `Project.automation_json.character_personality` 读取 strict gate、fallback、cast diversity、relationship enrichment、low confidence blocking 等项目策略。
- API 已提供 create、preview、coverage、backfill、reassign、character assignment-report、assignment_id explain、active context preview、relationship enrichment、metrics。
- legacy import、SubWorld core named character、planned roster materialization、BookState GraphDelta create character、arc plan seed character 已接入 helper。
- `tests/test_no_bare_character_creation.py` 静态约束会阻断 helper 外新增 `create_entity(kind="character")` 和 helper 外裸持久化 `WorldNode(node_type="character")`。
- context assembler 已接入 personality integrity gate：
  - `creation_status="legacy"` 默认 non-strict，只在 `ChapterContextPack.personality_integrity_issues` 暴露问题。
  - 非 legacy project 默认 strict，或由 `Project.automation_json.character_personality.strict_integrity` 显式控制。
  - strict 失败会写 `CHARACTER_INTEGRITY_CHECK_FAILED` 并阻断 writer context assembly。
- reassign 已输出 old/new loadout diff，并把 diff 写入 `PERSONALITY_LOADOUT_REASSIGNED` payload。
- `RelationshipPersonalityEnricher` 已接入 legacy `RelationEdge`，可把 rival/对手/竞争 映射到 `rel-rival-respect`，mentor/导师/师父/扶持/保护 映射到 `rel-mentor-protector`；默认不覆盖 manual override，不重复添加同 target pattern，diff 写 `PERSONALITY_RELATIONSHIP_ENRICHED`。
- `rel-rival-respect` 与 `rel-mentor-protector` 已补最小运行时内容与 prompt compression，不再作为占位 skill 注入 runtime。
- deterministic `PersonalityConsistencyReviewer` 已把 missing loadout、reference model override、untriggered stress behavior 等可检测子集作为 personality lint signals 合入 `HistoricalReviewHub`。
- metrics API 已输出 creation、auto assignment、manual override、fallback、low confidence、missing loadout、confidence avg、OOC issue counts 与 dominant skill distribution。
- coverage API 已输出 `issue_counts`、`skill_distribution`、`personality_ooc_issue_counts` 和 per-character issues，并支持 dashboard 所需筛选键。
- World Studio 人物性格页已增加人物创建表单、自动预览、可编辑 loadout JSON、assignment report view、reassign、active context preview、relationship enrichment 与 metrics summary。

仍无法完全自动化或保持边界的部分：

- LLM-based OOC 深度诊断仍不在本增量内；当前验收目标是 deterministic reviewer checks 加现有 reviewer evidence/prompt 集成。
- metrics 是项目级 API/dashboard aggregate，不是 Prometheus exporter。
- relationship enrichment 默认只自动使用 `rel-rival-respect` 与 `rel-mentor-protector`；其他 relationship pattern skill 保持 catalog 可见但不自动选择。
- reviewer 只能检测文本中可确定识别的子集，无法在没有 scene/canon evidence 的情况下自动证明所有 OOC 归因。
