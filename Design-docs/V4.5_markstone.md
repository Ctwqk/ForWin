# ForWin V4.5 Markstone：设计文档收束与实现差距

更新时间：2026-04-26

状态：设计收束文档；用于替代旧 V2/V3 历史设计索引，作为当前代码与设计差距的统一入口。

## 1. 定位

V4.5 不重新发明一套架构，而是把当前代码、保留设计文档和已落地实现对齐到一个判断口径：

```text
Genesis / Writer / Review 主链
+ Skill Runtime
+ Observability
+ Final BookState Runtime
+ Scheme C BookMap
= V4.5 当前工程基线
```

当前最终方向：

- 书本根真值来自 Genesis。
- 章节生产仍由 `Arc -> Band -> Chapter -> Scene` 主链驱动。
- 旧 `world_model_v4` 仍在主链中运行，但最终 canon 方向是 `BookState + GraphDelta + Snapshot`。
- 地图最终语义是 `BookMap = SubWorld -> Region -> MapNode -> MapEdge`。
- `SubWorld` 不再表示城市、宗门外门、客栈、遗迹入口等局部舞台。
- 观测、审计、PromptTrace、ArtifactStore 是主链可维护性的底座，不是额外调试代码。

## 3. 当前代码实况

### 3.1 已落地

- Genesis 根层、阶段生成、阶段锁定、`start-writing` handoff 已存在。
- Writer 已支持 scene-aware 生成、fallback、structured output、review/repair/canon 主流程。
- Candidate Draft、Scenario Rehearsal、checkpoint、task-center、governance 和 DecisionEvent 主链已存在。
- Skill Runtime 已有 `forwin/skills/`，并接入 Genesis、Writer、Reviewer 的 prompt layer 与 PromptTrace。
- Observability 已有 `DecisionEvent`、`PromptTrace` 详情、chapter ledger、ArtifactStore、运行事件记录入口。
- Final BookState 已有 protocol、ORM、runtime、repository、projection、compiler、legacy import、review gate 和 debug API。
- `WritingOrchestrator` 已在旧 V4 compiler commit 后追加 BookState review/compile gate。
- Scheme C 地图包 `forwin/map/` 已落地：models、protocol、repository、generator、pathfinding、validator、service。
- BookMap 可通过 `create_or_update_book_map()` 生成多个 subworld，并用 `world_gate` 跨 subworld 连通。
- Reviewer context 已携带 `map_context`，heuristic reviewer 能检查跨 scene 地图移动不可达或 travel time 超时。

## 4. 设计文档与代码差距

### 4.1 `V2_9_2.md`

已对齐：

- Genesis 作为创建书本阶段的根层输入。
- `world` 根对象、map/story engine 数据、文化命名、region/node/faction/cast 字段已经大体进入代码。
- Writer / review / repair / governance 主链存在。
- Scenario Rehearsal 和 Candidate Draft 语义已进入当前代码。

差距：

- 文档仍保留大量 V2.9 的旧 `SubWorld` 局部舞台语义；当前代码已升级为 Scheme C 大尺度地图容器。
- Genesis `map_atlas` 与新 `map_regions / map_nodes` 之间还没有完整 promotion / merge UI。
- V2.9.2 对 experience layer、audience hint、review repair 的描述比当前 deterministic reviewer 约束更宽。
- `SubWorldPlanItem.region_seeds[] -> map_regions` 的迁移语义已实现方向，但还没有完整历史项目迁移工具和审计 UI。

处置：

- `V2_9_2.md` 只保留 Genesis / Writer / Review / Governance 基线语义。
- 地图与 SubWorld 语义以 `map_scheme_c.md` 为准。

### 4.2 `V2_9_3_skill_runtime.md`

已对齐：

- `forwin/skills/` 已存在 loader、registry、router、prompt layer。
- Genesis、Writer、Reviewer 已能选择 skill layers。
- `PromptTrace` 已记录 `selected_skills` 和 `skill_summary`。

差距：

- 仍未做 Skill API / UI。
- 仍未做项目级 skill pinning。
- 仍未做脚本型 Skill。
- Skill strictness 与启用/禁用策略还偏配置级，不是项目治理级。

处置：

- V4.5 暂不扩大 Skill Runtime 范围；先保证 prompt layer 可追溯和不绕过 canon gate。

### 4.3 `V3_8.md`

已对齐：

- `DecisionEvent` 已作为业务审计事件账本。
- `PromptTrace` 已可通过 API 查询详情。
- `ArtifactStore` 已用于 frozen candidate、writer output 等大对象。
- Chapter ledger API 已存在。

差距：

- runtime JSON logging、metrics、redaction、trace completeness 仍不是全链路强制。
- observability 还没有统一 SLO / dashboard。
- BookState gate、Scheme C map generation 和 map reviewer issue 还需要更细粒度 trace event。
- API 返回仍偏 debug shape，未做稳定外部契约。

处置：

- V4.5 后续应把 BookState compile、map generation、cognition path check 纳入 V3.8 观测标准。

### 4.4 `V4_final_book_state_runtime.md`

已对齐：

- `forwin/protocol/book_state.py`、`forwin/models/book_state.py`、`forwin/book_state/` 已落地。
- 已有 `ObjectiveWorldGraph`、`MapGraph`、`CognitionView`、`NarrativeControlGraph`、`BookStateRuntime`。
- 已有 `BookStateRepository`、`BookStateProjection`、`BookStateCompiler`、`LegacyBookStateImporter`。
- 已有 `BookStateReviewGate` 和 `BookStateDeltaAdapter`。
- 已有 BookState debug API：runtime status、map path query、legacy import。
- Orchestrator 已在 V4 compiler 后追加 BookState review/compile gate。

差距：

- 旧 `world_model_v4` 仍是主链里的第一层 review/compiler，BookState 还是追加 gate，不是唯一 canon source。
- Writer / extractor 还没有直接产出完整 `GraphDelta`；当前主要靠 adapter 从 V4 change set 转换。
- BookState snapshot / replay 已有基础，但还没有成为所有 context assembly 的唯一来源。
- World Studio 还没有展示 BookState graph、map path、cognition refs、delta ledger。
- API 没有稳定 response model，仍偏 debug endpoint。
- BookState reviewer 仍是 deterministic guardrail，不是完整 world/map/cognition rule pack。

处置：

- V4.5 的核心工程目标是把 BookState 从追加 gate 推进为主 canon runtime。

### 4.5 `map_scheme_c.md`

已对齐：

- `SubWorld` 大尺度地图容器字段已加入 ORM/migration。
- `map_regions`、`map_region_edges`、`map_generation_runs` 已加入。
- `map_nodes` / `map_edges` 已扩展 subworld/region/description/weight 字段。
- `SubWorldMapSpec`、`MapAnchorNodeSpec`、`BookMapRuntime`、`MapValidationReport` 等协议已落地。
- Scheme C 生成器已支持 deterministic seed、required regions、anchor nodes、MST、extra edges、edge weights、validation。
- Dijkstra、metric、多重边、有向边、bidirectional reverse、hidden/blocked、observer cognition overlay、false edges 已有测试覆盖。
- `create_or_update_book_map()` 支持跨 subworld `world_gate`。
- reviewer 已能用 `map_context` 做移动时间检查。

差距：

- 没有独立 FastAPI map service；当前只有 Python service 和 BookState path debug API。
- Map generation 尚未自动接入 Genesis `start-writing` 或 arc expansion。
- `site_state` 与 `MapNode` 的自动双层生成还没有完整工具。
- 跨 subworld 连接目前是 service-level 默认相邻连接，不是完整 SubWorldMetaGraph planner。
- A* 只在旧 BookState `MapGraph` 中有基础 heuristic；Scheme C service path 仍以 Dijkstra 为主。
- reviewer 只做最小移动合理性检查，还没有完整 hidden route 越权、blocked route、时间窗口和 observer-known route rule pack。

处置：

- V4.5 应先把地图生成接入 Genesis / orchestrator，再扩 reviewer rule pack。

### 4.6 `writing_flow_state_machine.md`

已对齐：

- 当前仍以 `start-writing`、`continue-generation`、task pause/terminate、review/repair/canon 为主状态机。
- Scenario Rehearsal 和 Candidate Draft 已进入主链。

差距：

- 文档尚未补入 BookState review/compile gate 在 canon commit 后的新增阻断点。
- 文档尚未补入 map movement issue 可能导致 review fail 的路径。

处置：

- 后续应在状态机图中新增 `BookStateReviewGate` 与 `MapMovementReview` 节点。

### 4.7 `provisional_mechanism_check.md`

已对齐：

- legacy provisional 已降级为 compatibility。
- Scenario Rehearsal / Candidate Draft / Canon Commit 分层与当前代码一致。

差距：

- 文档仍偏 4.1 视角，未包含 BookState final runtime 与 Scheme C map gate。

处置：

- 保留作为 legacy provisional 边界说明，不作为 V4.5 总架构入口。

## 5. V4.5 里程碑

### M1：设计与文档收束

目标：

- 保持当前文档集最小化。
- 任何旧 `SubWorld` 局部语义都只作为 compatibility 出现。
- 新设计入口固定为本文件、`V4_final_book_state_runtime.md`、`map_scheme_c.md`。

验收：

- 旧 V2/V3 历史设计稿不再出现在 `Design-docs/`。
- 活跃文档不再把城市、宗门、遗迹入口当作 `SubWorld`。

### M2：BookState 成为主 canon runtime

目标：

- Writer/extractor 直接产出或稳定转换为 `ApprovedGraphDeltaSet`。
- BookState compiler 成为 canon commit 的主路径。
- 旧 V4 world model 降级为 compatibility projection / migration source。

验收：

- Accepted chapter 后能从 BookState snapshot/replay 重建 world/map/cognition runtime。
- Context assembly 优先读取 BookState。
- 旧 `entities / entity_states / relation_edges` 只作为兼容投影或迁移来源。

### M3：Scheme C 地图接入主链

目标：

- Genesis map seed 可生成 `SubWorldMapSpec`。
- `start-writing` 或 arc initialization 能创建初始 BookMap。
- arc expansion 能创建新 subworld 或 region，并通过 `world_gate` / portal / route 接入 BookMap。

验收：

- 新项目可自动生成至少一个 subworld map。
- writer context 可拿到主角当前位置、region、nearby nodes、reachable nodes。
- reviewer 可用地图 travel time 检查章节移动。

### M4：Cognition-aware reviewer

目标：

- reviewer 区分 objective path 和 observer-known path。
- hidden route 未发现时不能被角色直接使用。
- false edges 可导致错误认知路径，但不能污染 objective map。

验收：

- 同一移动可同时给出 objective reachable 与 known unreachable / detour。
- hidden route、blocked route、false route 均有 deterministic issue code。

### M5：Observability hardening

目标：

- BookState compile、map generation、pathfinding review、cognition override 都写入可追踪 ledger。
- PromptTrace / DecisionEvent / ArtifactStore 形成统一调试路径。

验收：

- 单章失败能从 task timeline 定位到 review、map、BookState 或 LLM 层。
- BookState / map debug API 返回稳定 contract。

### M6：Migration and cleanup

目标：

- legacy SubWorld region drafts 提升为 `map_regions`。
- 旧 world_model_v4 rows 可导入 BookState。
- 旧局部 SubWorld 项目能给出迁移报告。

验收：

- 迁移不会把城市/宗门/遗迹误建为 subworld。
- validation report 能列出需要人工处理的 legacy scope conflict。

## 6. 当前非目标

V4.5 不做：

- 不做真实地理板块模拟。
- 不做 tile renderer。
- 不把 Neo4j 作为主存储。
- 不把地图路线塞进普通 `WorldEdge`。
- 不让 Writer 直接看到 hidden objective truth。
- 不把 Skill Runtime 扩成脚本执行平台。
- 不把 debug API 宣称为外部稳定 API。

## 7. 当前风险

- BookState 与旧 V4 compiler 双轨并存，短期会增加 canon commit 阻断点。
- `V2_9_2.md` 仍有历史兼容语义，开发时必须以 V4.5 / Scheme C 覆盖解释。
- 地图生成已经可用，但主链尚未自动调用，容易出现“有地图系统但项目没有地图数据”的空运行状态。
- reviewer 的 map check 已经能发现 travel time 问题，但还不能完整处理剧情时间窗口、交通工具、队伍速度、access rule 和 observer-known route。
- BookState debug API 已接入路由，但 response model 还未固化。

## 8. 执行口径

后续开发默认遵守：

- 读当前设计先读 `V4.5_markstone.md`。
- 世界状态最终以 `V4_final_book_state_runtime.md` 为准。
- 地图最终以 `map_scheme_c.md` 为准。
- Genesis / Writer / Review 主链仍参考 `V2_9_2.md`，但其中旧 `SubWorld` 语义不得作为新实现依据。
- 任何新迁移、API 或 reviewer 变更都必须更新 `maintenance_log.md`。
