# ForWin V4.6 知识系统设计文档

更新时间：2026-04-26

状态：设计落档。V4.6 将小说世界状态升级为三层知识系统：BookState DB Canon、Obsidian Vault、Karpathy-style LLM KB。合并与冲突处理必须遵守：DB canon 是唯一权威真相；Obsidian 和 LLM KB 都是可重建投影，不直接写 canon。

## 1. 核心定位

ForWin V4.6 将小说世界状态升级为三层知识系统：

```text
BookState DB Canon = 唯一权威真相
Obsidian Vault = 人类可读、可编辑、可审查的知识网投影
Karpathy-style LLM KB = LLM 可读、可重建、角色隔离的动态工作记忆
```

权威关系固定为：

```text
DB canon 负责真相
Obsidian 负责人类理解
LLM KB 负责模型上下文
```

Obsidian 和 LLM KB 都不是 canon，均可重建、覆盖、丢弃。

## 2. 总体架构

```text
BookState DB Canon
  ├─ ObjectiveWorldGraph
  ├─ MapGraph
  ├─ CognitionOverlay
  ├─ ReaderExperienceState
  ├─ GraphDeltaLedger
  └─ MaterializedSnapshot

        ↓ export / projection

Obsidian Vault

        ↓ compile / role scope

Karpathy-style LLM KB
  ├─ writer-safe root files
  ├─ JSONL facts/events/deltas/questions
  ├─ role packs
  └─ vector retrieval index
```

现有 `world_model` / `world_model_v4` 保留为兼容层和迁移来源。V4.6 的新投影与上下文系统以 `BookState` 为 canon source。

## 3. BookState Canon

BookState 是 V4.6 的权威世界模型，包含：

- `WorldNode`：角色、势力、组织、家族、地点、资源、规则、制度、技术、事件、秘密、知识缺口、读者承诺、冲突、写作约束等。
- `WorldEdge`：控制、拥有、效忠、敌对、亲属、师徒、因果、约束、承诺、兑现、揭示、误导、伏笔等关系。
- `FactNode`：结构化事实。
- `MapNode / MapEdge`：独立地图拓扑，支持路径、隐藏路线、封锁路线和 observer filtering。
- `CognitionOverlay`：读者、角色、势力等观察者的认知 mask、false belief、visibility override。
- `ReaderPromise / ReaderExperienceDelta`：读者承诺、爽点债务、payoff、情绪效果。
- `GraphDelta`：所有 canon 变化的追加式 ledger。
- `MaterializedSnapshot`：按章节重建后的快照。

核心原则：

```text
章节文本不能直接写 canon
LLM 输出不能直接写 canon
人工修改不能直接写 canon
所有进入 canon 的变化必须经过 BookStateReviewGate
BookStateCompiler 只接受 approved GraphDeltaSet
```

## 4. GraphDelta 流程

每次章节 accepted 后：

```text
Chapter accepted
  ↓
WorldChangeExtractor
  ↓
V4 ReviewGate
  ↓
WorldModelCompilerV4
  ↓
BookStateDeltaAdapter
  ↓
BookStateReviewGate
  ↓
BookStateCompiler
  ↓
GraphDelta append
  ↓
Snapshot rebuild
  ↓
Projection refresh
```

`BookStateCompiler` 支持：

- `create`
- `set`
- `merge`
- `append`
- `remove`
- `deactivate`

并基于 `old_value` 检测冲突，避免旧上下文覆盖新 canon。

## 5. Obsidian 投影系统

Obsidian 是人类知识网，不是数据库。

导出结构：

```text
00_Index.md
01_Book/
02_Map/
03_Actors/
04_Systems/
05_Plot/
06_Secrets/
07_Reader/
08_Conflicts/
09_LLM_KB/
```

每个页面包含稳定 frontmatter：

```yaml
forwin_id:
node_id:
node_type:
status:
as_of_chapter:
visibility:
truth_relation:
source_refs:
locked_fields:
editable_fields:
```

页面分区：

```text
Canon Summary        系统生成，锁定
Current State        系统生成，锁定
Relationships        系统生成，锁定
Reader Visibility    系统生成，锁定
Open Questions       系统生成，锁定
Evidence             系统生成，锁定
Manual Notes         人类可编辑
Human Questions      人类可编辑
Proposed Correction  人类可编辑
```

Import 规则：

```text
Obsidian 修改不能直接改 canon
locked section 修改只生成 proposal
editable section 修改生成 proposal
删除页面不删除 canon
proposal approve 后转 GraphDelta
proposal reject 只记录状态
```

结构化修正支持 `forwin-patch` fenced JSON，例如：

````md
```forwin-patch
[
  {
    "op": "set_node_field",
    "node_id": "char_lin",
    "field_path": "summary",
    "old_value": "旧摘要",
    "new_value": "新摘要"
  }
]
```
````

## 6. LLM KB

LLM KB 是 Karpathy-style 动态知识库，面向模型上下文，不保存唯一真相。

输出目录：

```text
data/llm_kb/{project_id}/
  CURRENT_STATE.md
  NEXT_CHAPTER_CONTEXT.md
  ACTIVE_THREADS.md
  CHARACTER_MEMORY.md
  FACTION_MEMORY.md
  MAP_CONTEXT.md
  READER_PROMISES.md
  KNOWLEDGE_GAPS.md
  REVEAL_LADDER.md
  MUST_NOT_REVEAL.md
  RECENT_CHANGES.md
  STYLE_AND_TONE.md
  CONSTRAINTS.md

  facts.jsonl
  events.jsonl
  graph_deltas.jsonl
  open_questions.jsonl
  retrieval_index.json
  packs/{writer|reviewer|planner|compiler}/context.json
```

root 文件默认 writer-safe：

```text
不暴露 hidden objective truth
不暴露未来反转完整解释
MUST_NOT_REVEAL 只列 forbidden refs，不泄露秘密正文
```

角色权限：

```text
WriterPack   只看可写出的公开事实、当前角色认知、禁止揭示列表
ReviewerPack 可看更多客观真相，用于检查一致性
PlanningPack 可看长期结构、承诺债务、future constraints
CompilerPack 可看完整 canon delta、old/new value、review verdict
```

LLM KB 也包含本地 vector index，支持按 role scope 检索。

## 8. 安全边界

V4.6 的关键安全约束：

```text
1. DB canon 是唯一真相。
2. Obsidian 不是 canon。
3. LLM KB 不是 canon。
4. proposal approve 必须走 ReviewGate。
5. old_value mismatch 会阻断。
6. writer root 不暴露 hidden truth。
7. reviewer/compiler 可看更多 truth。
8. 所有 canon 变化可 replay、diff、audit。
```
