# ForWin V4.7 Character Personality Skill 维护文档

更新时间：2026-04-27

状态：维护规则落档。本文档用于维护 `forwin_skills/character_personality/`、BookState `personality_loadout`、writer/reviewer personality runtime，以及 World Studio 人物性格编辑入口。

## 1. 维护对象

V4.7 维护对象分四类：

```text
skill 内容维护
  -> forwin_skills/character_personality/**

runtime 维护
  -> forwin/personality/**
  -> forwin/writer/prompts.py
  -> forwin/reviewer/webnovel.py
  -> forwin/context/assembler.py

BookState / API / UI 维护
  -> character WorldNode.profile.personality_loadout
  -> /api/personality-skills
  -> /book-state/characters/*/personality-loadout
  -> frontend/world-studio

测试维护
  -> tests/test_personality_runtime.py
  -> tests/test_world_studio_frontend.py
```

## 2. Skill 内容维护原则

每个 skill 必须遵守：

- 不写具体角色 canon。
- 不写 `character_id / character_name / current_goal / scene_context`。
- 不用临床诊断标签作为主 skill 名。
- 不从 MBTI、Enneagram、DISC、Socionics、archetype label 直接推导行为。
- 至少包含 `Body Language`。
- 至少包含 `Prompt Compression`。
- `Do Not` 必须包含 canon 优先与 reference model 降级规则。

## 3. 新增 Skill 流程

1. 在 `catalog/skill_catalog_mapping.md` 增加条目。
2. 在对应目录创建：

```text
skills/{type}/{skill-name}/SKILL.md
skills/{type}/{skill-name}/examples.md
skills/{type}/{skill-name}/test_cases.md
```

3. 使用对应模板：

```text
templates/SKILL_TEMPLATE_TRAIT.md
templates/SKILL_TEMPLATE_SOCIAL_MASK.md
templates/SKILL_TEMPLATE_STRESS_MODE.md
templates/SKILL_TEMPLATE_RELATIONSHIP_PATTERN.md
templates/SKILL_TEMPLATE_ARCHETYPE.md
```

4. 确认 frontmatter 至少包含：

```yaml
name:
chinese_name:
description:
forwin_scope: character_personality
category: character_personality_skill
skill_type:
version:
status:
mapping_confidence:
mode: instruction_only
reference_model_policy:
tags:
```

5. 确认正文至少包含：

```text
Core Function
Runtime Priority
Decision Mechanics
Relationship Pattern
Expression
Body Language
Stress and Arc
Do Not
Prompt Compression
```

6. 运行验证：

```bash
uv run --extra test python -m pytest tests/test_personality_runtime.py -q
uv run python -m json.tool forwin_skills/character_personality/schema/personality_skill.schema.json >/dev/null
```

## 4. 修改 Skill 流程

修改 skill 时先判断修改类型：

### 4.1 文案增强

只增强表达、例子、中文描述，不改变 runtime 行为。

要求：

- 不需要迁移角色 loadout。
- 需要确认 `Prompt Compression` 没有无意改变。
- reviewer 结果可能变化，需跑 personality runtime 测试。

### 4.2 行为规则改变

改变 trigger、relationship、stress、growth 或 prompt compression。

要求：

- 更新对应 `test_cases.md`。
- 检查已使用该 skill 的角色 loadout 是否需要 override。
- 对已生成章节，不能 retroactively 改 canon，只能影响后续写作。
- 在维护日志写明行为变化。

### 4.3 skill 重命名

高风险操作。

要求：

- 优先避免重命名。
- 如果必须重命名，需要提供 alias 或迁移脚本。
- 更新所有 character `personality_loadout`。
- 更新 catalog、compatible_with、tension_with、tests。

## 5. Prompt Compression 维护

runtime 优先读取 `## Prompt Compression` fenced YAML。

字段建议：

```yaml
prompt_compression:
  one_line_summary:
  perception_bias:
  decision_bias:
  dialogue_bias:
  body_language_bias:
  relationship_bias:
  stress_bias:
```

social mask 可额外包含：

```yaml
outward_bias:
hidden_bias:
crack_bias:
```

stress mode 可额外包含：

```yaml
trigger_bias:
recovery_bias:
```

维护规则：

- 每条 bias 应该短、可直接进入 prompt。
- 不要把整段心理学说明塞进 compression。
- 不要把 reference model label 放进 compression。
- 不要写具体角色名字或 canon。

## 6. Loadout 维护

角色 loadout 写在 `WorldNode.profile.personality_loadout`。

修改入口：

- World Studio 人物性格页。
- API `PUT /book-state/characters/{character_id}/personality-loadout`。

维护规则：

- dominant 最多一个。
- secondary 可以多个，但应控制数量。
- social_mask 必须有清晰 `active_when`，除非这个 mask 是角色默认公共形象。
- stress_modes 必须有 trigger。
- relationship_patterns 必须有 target，除非它是角色的全局关系模式。
- overrides 只写角色特定调参，不复制整份 skill。

## 7. Reviewer 维护

Reviewer 应检查：

- `personality_ooc`：行为无法由 active context、canon、relationship 或 stress 解释。
- `tendency_as_absolute`：把倾向写成绝对规则。
- `reference_model_override`：从模型标签直接推导行为。
- `stress_mode_without_trigger`：压力模式无 trigger 常驻。
- `social_mask_as_self`：社交面具覆盖了人物本体。
- `growth_erases_mechanism`：成长把人格机制删除，而不是健康化。
- `skill_invented_canon`：skill 发明新事实。

新增 reviewer 检查时必须保证：

- reviewer 不直接加载完整 skill 仓库。
- reviewer 只使用 `active_personality_context`。
- issue 必须引用 evidence id。

## 8. World Studio 维护

当前 UI 是低风险 JSON 编辑器，后续可升级为结构化编辑器。

维护规则：

- 不在前端硬编码完整 skill 内容。
- catalog 从 `/api/personality-skills` 读取。
- 保存前保持 JSON 可解析。
- 后端负责校验 skill id 是否存在。
- 保存成功后刷新当前角色 loadout。

后续结构化 UI 建议：

- dominant trait select
- secondary multi-select
- social mask active_when editor
- stress mode trigger editor
- relationship pattern target editor
- compatible/tension warning
- active context preview

## 10. 读者 OOC 反馈处理

收到“人物 OOC”反馈时按顺序定位：

1. canon 是否已经支持这次行为。
2. scene goal 是否强制该行为。
3. relationship state 是否变化。
4. stress trigger 是否激活。
5. active context 是否包含对应 bias。
6. writer 是否忽略 active context。
7. skill 本身是否缺少规则。
8. loadout 是否缺少 secondary / mask / relationship pattern。

处理结果分四类：

- 修章节：写作输出误用。
- 修 loadout：角色配置不完整。
- 修 skill：通用机制缺漏。
- 修 canon/relationship state：运行时状态没有反映剧情变化。

每次修复应补一个 `test_cases.md` 或 pytest 回归。

## 11. 发布与部署

纯 skill 内容或设计文档修改：

- 不需要重启服务。
- 不需要部署到 `8899`。
- 但如果线上服务需要读取新 skill 文件，需要按部署流程同步文件。

runtime/API/UI 修改：

- 需要跑测试。
- 需要确认没有 active generation。
- 需要按 ForWin 维护流程重建并切换服务。

部署前检查：

```text
无 active generation
personality tests pass
skill runtime tests pass
frontend build pass if UI changed
maintenance_log 已记录影响与验证
```
